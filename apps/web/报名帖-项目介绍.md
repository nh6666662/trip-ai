### 1. 创意名称 + 创意介绍

**创意名称**：旅智 TripAI

**想解决什么问题**：旅行者花大量时间在小红书、马蜂窝、Google Maps 之间反复横跳——查景点、排路线、算时间、订餐厅。好不容易定好的计划,一到现场全乱：突降暴雨、景点排队两小时、餐厅关门、交通延误。规划难、变化多、信息滞后,是旅行三大顽疾。

**为什么会想到做这个**：在一次去杭州的旅行中,早上 9 点到断桥发现人山人海,临时改去苏堤却找不到附近吃饭的地方,下午又突降阵雨打乱了所有户外安排。那一刻我问自己：能不能让 AI 像本地朋友那样,提前告诉我什么时候去哪、下雨怎么办、人多了去哪？

**大概是什么产品**：Web 控制台 + 桌面应用 + REST API,前后端分离架构。前端 Next.js + React + Framer Motion,后端 Supabase + 高德地图 API + 多 LLM 接入（OpenAI 兼容协议）。AI 生成行程,Supabase Realtime 推送预警,高德提供周边搜索与地理围栏。

**一句话概括**：旅智 TripAI 不是一个行程生成器,而是一个会"陪你看天气、躲人流、找厕所"的 AI 旅行管家。规划只是起点,陪伴全程才是核心。

---

### 2. 目标用户及痛点

**面向哪些用户**：自由行旅客、家庭出行者、独立旅行者、5 人以下小团队。他们需要个性化的行程推荐,但又不想花 3 天查资料;想要说走就走的自由,但又怕现场翻车。典型场景：一个人去陌生城市旅行,既要规划又要应变,既要省心又不想跟团。

**在什么场景下使用**：

- **行前规划**：输入目的地、天数、人数、风格（紧凑版/松弛版）,AI 30 秒生成可执行行程,带交通方式、停留时长、备选方案。
- **行中应变**：突降暴雨、景点排队、餐厅关门,实时预警 + 备选方案一键采纳,自动替换原节点。
- **全程陪伴**：午餐吃什么（高德扫街榜按餐品类型推荐）、厕所在哪（最近公共厕所标记）、累了去哪歇（AI 推荐附近咖啡馆/商场/公园）。
- **社区回流**：旅行结束后上报真实体验,数据回流模型,让下一次推荐更懂真实世界。

**当前痛点（三层减负）**：

| 层面 | 以前怎么扛 | 用了之后 |
| --- | --- | --- |
| **规划层** | 小红书、马蜂窝、Google Maps 反复横跳,3 天只排出 1 个草案 | 输入目的地+天数+偏好,AI 30 秒生成可执行行程,紧凑/松弛任选 |
| **应变层** | 突降暴雨、景点排队 2 小时,现场手忙脚乱查替代方案 | 天气/人流/交通实时预警,AI 推荐备选方案,一键采纳替换节点 |
| **决策层** | 景点门口随便进一家被宰,找不到厕所憋一路 | 高德扫街榜推荐餐厅、最近公共厕所、AI 休息点推荐,全程陪伴 |

**核心差异**：把"人查机器"变成"机器陪人"——AI 不只是生成一份 PDF,而是全程实时陪伴。

**⚠️ 但它不是又一个行程生成器**：传统工具只解决"规划层"——给你一份攻略就结束了。旅智 TripAI 的核心是**实时应变层**：基于 Supabase Realtime 推送 + 多源数据交叉验证（天气 API + 高德人流 + UGC 上报）,让行程在旅途中持续进化。规划只是起点,陪伴全程才是差异。

---

### 3. 价值与意义

**⚡ 效率提升**：三个层面同时减负——

- **规划层**：从 3 天查资料缩到 30 秒生成,AI 做掉 70% 的机械劳动
- **应变层**：不用现场手忙脚乱,实时预警 + 备选方案一键采纳
- **决策层**：不用纠结吃什么去哪歇,高德扫街榜 + AI 休息点推荐全程陪伴

旅行从"人查机器"变成"机器陪人",预计规划时间减少 70%,旅途翻车率降低 80%。

**📈 商业价值与想象力**：旅行规划只是第一个用例,**AI 陪伴引擎**才是底子。这套"意图理解 + 实时应变 + 社区回流"的能力不绑定旅行领域——

- **本地生活**：周末去哪、约会路线、亲子活动
- **商务出行**：会议行程、客户拜访路线
- **研学旅行**：教育路线、学习任务编排
- **银发旅行**：适老化陪伴、健康提示、就近休息

相比纯工具的窄赛道,AI 陪伴引擎的市场天花板高一个数量级。

### 4. 核心能力矩阵

| 能力 | 技术实现 | 差异化 |
| --- | --- | --- |
| **AI 行程生成** | Next.js + Supabase + 多 LLM 接入（OpenAI 兼容协议） | 30 秒生成可执行时间轴,带交通方式、停留时长、备选方案 |
| **实时预警** | Supabase Realtime + 天气 API + 高德人流 | 三源交叉验证,置信度 ≥ 0.7 才推送,避免信息过载 |
| **备选方案采纳** | React Query invalidateQueries + 节点 PATCH | 一键采纳替换原节点,动画过渡,预警自动移除 |
| **AI 关心小贴士** | `/api/ai/care-tip` LLM 流式生成 | 忽略预警后 AI 根据内容生成 15 字 tip,写入对应节点 |
| **高德周边搜索** | `/api/places/around` + `/api/places/search` | 餐厅按餐品类型筛选、公共厕所按距离排序、AI 休息点推荐 |
| **天气背景动画** | 纯 CSS GPU 加速（替代 Framer Motion JS） | 晴天光环旋转/雨天雨滴下落/雪天雪花飘落,覆盖整个头部卡片 |
| **UGC 社区分享** | Supabase RLS + 地理围栏验证 | 用户上报真实体验,数据回流模型,越用越懂真实世界 |

### 5. 技术栈

```
前端：Next.js 14 + React 18 + TypeScript + Tailwind CSS + Framer Motion
后端：Supabase (Auth + PostgreSQL + Realtime) + Next.js API Routes
AI：多 LLM 接入（OpenAI 兼容协议,支持 Agnes / LongCat / 自定义）
地图：高德地图 JS API + 周边搜索 + 地理围栏
状态：Zustand + React Query + Supabase Realtime
部署：Vercel + Supabase Cloud
```
